# Agent Skills（エージェント・スキル）完全解説：AIを「専門家」に変える新しい標準

AIコーディングアシスタント（AIエージェント）に毎回「この言語を使って」「この手順でテストを書いて」と指示することに疲れていませんか？

AIの性能が向上する一方で、プロジェクト固有の複雑なワークフローを毎回AIに理解させる「手動プロンプト」の限界が見えつつあります。この課題を抜本的に解決し、現在AI開発のトレンドとして最も注目を集めているのが **「Agent Skills（エージェント・スキル）」** です。

本ドキュメントでは、「Skillsとは何か」「どのような利点があるのか」「どう使うのか」、そして最近話題の「skills.sh」について、初学者が本質を理解できるように解説します。

## 1. なぜ今「Agent Skills」が注目されているのか

これまでのAIエージェント（Claude Code、Cursorなど）は、いわば「非常に優秀なインターン生」でした。何でもそこそここなせますが、**「仕事の具体的な進め方（ワークフロー）」は持っていませんでした**。

そのため、開発現場では次のような問題が頻発していました。

- 「リサーチレポートを書いて」と頼むと、毎回違うフォーマットで出力される
- 「テストを書いて」と頼むと、プロジェクトの規約を無視したコードが生成される
- プロジェクトの規約や手順を「システムプロンプト」にすべて詰め込もうとすると、コンテキスト（文脈）が肥大化しすぎてAIが混乱する

単純なチャットやコード補完の時代が終わり、自律的に複数のタスクをこなす「エージェント型AI」の時代に入った今、**「AIに複雑な仕事を一貫して任せるための手順書」**が必要になりました。それが「Agent Skills」です。

---

## 2. Agent Skillsとは何か？（概念の解説）

**Agent Skills（スキル）** とは、分かりやすく言えば**「AIエージェントに専門知識と具体的な手順を教えるための、持ち運び可能なマニュアル」**のことです。

2024年末〜2025年にかけてAnthropic社（Claudeの開発元）が『[agentskills.io](https://agentskills.io)』としてオープン標準（誰もが利用できる共通仕様）を策定したことで一気に普及しました。

### 「新入社員」と「専門マニュアル」の比喩で理解する

- **素のAIエージェント**: 頭は良いが手順を知らない新入社員
- **Agent Skills**: 特定業務（例：足場の組み方、リリースノートの書き方）に特化した「専門マニュアル」

AIエージェントは、この専門マニュアル（スキル）を作業机（ディレクトリ）に置いておきます。そして、ユーザーから**「スライドを作って」と頼まれた瞬間だけ、該当するマニュアルを開き、その通りの手順で完璧に作業をこなす**ようになります。

## 3. Agent Skillsを導入する3つの利点

システムプロンプトや単なるルールファイル（Rules）と比較した場合、Skillsには決定的な3つのメリットがあります。

### ① オンデマンド読み込みによる「コンテキストの節約」
最も重要な利点です。これまでAIには「テストの書き方」「DBの設計ルール」「デプロイ手順」などのプロジェクト知識をすべて常時読み込ませる必要がありました（コンテキストウィンドウの浪費）。
しかしSkillsは、**そのタスクに関連する（= 必要になった）瞬間にだけ**読み込まれます。これによりAIの脳を圧迫せず、高いパフォーマンスを維持できます。

### ② プログレッシブ・ディスクロージャー（段階的な詳細開示）
AIは最初からスキルの「全文」を読み込むわけではありません。
1. まず、数行の「このスキルはいつ使うべきか（description）」だけを読んでおく。
2. ユーザーの指示が該当した時だけ、「詳細な手順書」や「実行スクリプト」をロードする。
この2段階構造により、プロジェクト内に何十個ものスキルを配置しておいてもAIがパンクしません。

### ③ ポータビリティ（持ち運びやすさと共有）
Agent Skillsはオープン標準で作られているため、特定のツールに依存しません（Vender Lock-inの回避）。
例えば、あなたがVSCode（GitHub Copilot）向けに作った「社内用コードレビュースキル」のフォルダを、別のチームメンバーのCursorやClaude Codeにコピーするだけで、同じように動作します。

## 4. 類似機能との違い：Rules、CLAUDE.md、MCPサーバー

AIエージェントに指示を与える設定は複数あり、それぞれ役割が異なります。Skillsが他とどう違うのか、ここで整理しておきましょう。

| 設定の種類 | 役割と読み込みタイミング | 分かりやすい比喩 | 具体例 |
| :--- | :--- | :--- | :--- |
| **CLAUDE.md / AGENTS.md** | プロジェクトの「基礎知識」。**毎回自動**で読み込まれる | 現場のルールブック・全体マップ | ビルドコマンド、ディレクトリ構成 |
| **Rules（ルール設定）** | AIの「振る舞いの制約」。**常時**すべての回答に適用される | 常時遵守の安全規則（ヘルメット着用等） | 「TypeScriptを使う」「コメントは日本語」 |
| **Skills（本題）** | 特定の複雑なタスクの「専門手順書」。**オンデマンド**（関連タスク時のみ） | 高所作業時のみ取り出す「足場の組み方マニュアル」 | 「レポートを書く手順」「テスト自動化手順」 |
| **MCPサーバー** | 外部サービスとの「リアルタイム接続」。**接続中**常にアクセス可能 | 本社資材センターとの直通電話回線 | Slack読み書き、Google Drive検索 |

### 要点

- **CLAUDE.md・Rules** は「常に守るべき静的なルールや背景知識」です。これらに複雑な手順を書きすぎると、コンテキストウィンドウが枯渇してしまいます。
- **Skills** は「動的な専門手順書」です。必要な瞬間にだけ取り出されるため、コンテキストを節約しつつ複雑なワークフローを実現できます。
- **MCPサーバー** はファイルを読み込む仕組みではなく「外部と通信するプログラム（回線）」なので、Skillsとは完全に別物であり、併用することでさらに強力になります。

## 5. エコシステムとしての「skills.sh」の実態

Agent Skillsの広がりに伴い、最近コミュニティで急激に注目を集めているのが `skills.sh` です。

### skills.shとは？
[skills.sh](https://skills.sh) は、主にVercel社（Next.jsの開発元）のメンバーらが中心となって進めている、Agent Skillsのための**「オープンソースのエコシステムおよびマーケットプレイス」**です。

1. **スキルの共有場所（マーケットプレイス）**
   NPMパッケージのように、世界中の開発者が作って公開した優良なSkillsを検索・ブラウズできるウェブサイトです。
2. **コマンドラインツール（ランタイム）**
   `npx skills.sh install mcp-server-postgres` のようなコマンド一発で、GitHub上にあるスキルを自分のプロジェクトに瞬時にインストールできる仕組みを提供しています。

### なぜ重要なのか？
これまでSkillsは「各自でテキストファイルを書くもの」でしたが、`skills.sh` の登場により「**世の中の優秀なエージェントの振る舞いを、パッケージとしてダウンロードして適用する**」文化が生まれつつあります。GitHub上にキュレーションされた良質なスキル群（`github/awesome-copilot` など）を簡単に導入できるハブとして機能しています。

*※注意：外部から取得したスキルを使用する際は、内容（特にスクリプト）にセキュリティ上のリスクがないか必ず目視で確認しましょう。*

## 6. スキルの仕組み：SKILL.mdの構造

では、実際のスキルはどのような構造になっているのでしょうか？
スキルの実体は、1つの**『フォルダ』**と、その中にある**『SKILL.md』**という1つのMarkdownファイルです。

### フォルダ構成の基本

```
.agents/skills/            ← AIツールが読みに行く大枠のフォルダ
└── report-writer/         ← スキル名のフォルダ（= 1つのスキル）
    ├── SKILL.md             ← 【必須】スキルの定義ファイル
    ├── scripts/             ← （任意）実行に必要なスクリプト
    └── references/          ← （任意）長文の参考ドキュメント
```

### SKILL.mdの書き方（最重要）

`SKILL.md` の中身は2つの要素から成ります。これを理解すれば誰でもスキルを作れます。

```markdown
---
name: report-writer
description: このスキルを使用する条件を書く（※AIがここを読んで判断する最重要項目）
license: MIT
---

# リサーチレポート作成スキル

## 1. はじめに
このスキルが呼ばれたら、以下の手順でユーザーと対話してください。

## 2. 実行手順
1. ユーザーに「リサーチテーマ」を質問する
2. 最低10件のソースを検索する
3. `references/format.md` の形式に従ってレポートを出力する
```

> 💡 **最大のコツ**
> メタデータ（`---`で囲まれた部分）の `description` の書き方がスキルの成功を左右します。「Research tool」のような曖昧な書き方ではなく、「**ユーザーが『レポートを書いて』『リサーチして』と指示した時に使用してください。単なるコードの検索には使用しないでください**」といった具合に、「いつ使い・いつ使わないか」を明確に書くことで、AIの誤作動を防げます。

## 7. AIツールごとの利用方法（チートシート）

スキルの書き方は同じですが、AIツールによって「作ったスキルフォルダをどこに置くか」と「どうやって発動するか」が少し異なります。

| AI開発ツール | スキルフォルダの設置場所 | 発動させる方法（トリガー） |
| :--- | :--- | :--- |
| **Claude Code** | `.claude/skills/` | 自動（文脈からAIが判断） |
| **Cursor** | `.cursor/skills/` または `.agents/skills/` | 自動（文脈からAIが判断） |
| **VSCode (Copilot)** | `.github/skills/` | 手動（チャット画面から明示的に選択）|
| **Antigravity** | `.agents/skills/` | 自動 ＋ 手動（`@[/スキル名]` とメンション）|
| **OpenAI Codex** | リポジトリ内の任意の場所 （`AGENTS.md`で指定）| 自動（文脈からAIが判断） |

*※Tips: 複数のAIツールを併用するチームの場合、`.agents/skills/` などの共通の場所に置き、各ツールの設定ファイルで参照先をエイリアス（リンク）させる運用も増えています。*

## 8. まとめ：今日から始めるAgent Skills

「AIエージェントにプロンプト（指示）を出す時代」から、「**Agent Skills（手順書）を渡して任せる時代**」へとパラダイムシフトが起きています。

Skillsの利点をまとめます：
1. **AIの文脈を圧迫せず、必要な時だけ呼び出される**（オンデマンド）
2. **複雑な手順を一度書けば、いつでも高品質で再現できる**（再現性）
3. **他の人や別のツールにも簡単に共有できる**（ポータビリティ）

### まず試してほしいこと

このプロジェクトの `skills-example/` ディレクトリ配下に、すぐに試せる2つの実践的なスキルを用意しました。

- **`find-skills`**: ユーザーの質問から最適なスキルを検索・提案してくれるスキル
- **`skill-creator`**: 新しい対話型スキルの作成や改善自体をAIにサポートさせる強力なスキル

もしあなたが Antigravity や Cursor をお使いなら、これらのスキルを `.agents/skills/` などの所定のディレクトリにコピー（またはシンボリックリンクを作成）し、チャット欄で以下のように入力して、動作を体験してみてください。

```text
@[/find-skills] Reactのパフォーマンスチューニングを支援してくれるスキルを探して
```

AIがあなたに毎回「どうすればいいですか？」と聞くのではなく、スキルという「仕事の型」を持って自律的に動き始めるのを実感できるはずです。
